DynamoDB
==========================
🔘 aws.dynamodb.pk.hash

Defines how the table's hash key is extracted from the record. By default. partition is used as the hash key. The maximum size of a partition with this configuration is 10Gb as per DynamoDB limits.

	 - Type: STRING
	 - Default: partition
	 - Importance: HIGH
	 - Required: false

🔘 aws.dynamodb.pk.sort

Defines how the table's sort key is extracted from the record. By default, it uses the record offset as sort key. This sort key reference is created from a record reference and optional alias name. If the alias name is absent, then the last field of the reference is used as the column name. 

	 - Type: STRING
	 - Default: offset
	 - Importance: HIGH
	 - Required: false

🔘 aws.dynamodb.max.retry.time.seconds

The maximum time upto which DynamoDB client will try writing records. The least recommended value is 5 seconds.

	 - Type: LONG
	 - Default: 5
	 - Importance: LOW
	 - Required: false

🔘 table.name.format

A format string for the destination table name, which may contain '${topic}' as a placeholder for the originating topic name.

	 - Type: STRING
	 - Default: ${topic}
	 - Importance: MEDIUM
	 - Required: false

🔘 behavior.on.null.values

How to handle records with a non-null key and a null value (i.e. Kafka tombstone records). Use option 'UPSERT' to upsert the tombstone records along with the key, option 'IGNORE' to skip processing of the tombstone records and option 'DELETE' to delete the corresponding tombstone record in DynamoDB table.

	 - Type: STRING
	 - Default: UPSERT
	 - Importance: MEDIUM
	 - Required: false

🔘 aws.dynamodb.instant.flush

If set to true, the connector will flush the buffer immediately after receiving each record.This is useful for low latency requirements but may impact throughput. Default is false.

	 - Type: BOOLEAN
	 - Default: false
	 - Importance: LOW
	 - Required: false
