==========================
Exactly Once Support
==========================
🔘 exactly.once.support

Permitted values are requested, required. If set to "required", forces a preflight check for the connector to ensure that it can provide exactly-once semantics with the given configuration. Some connectors may be capable of providing exactly-once semantics but not signal to Connect that they support this; in that case, documentation for the connector should be consulted carefully before creating it, and the value for this property should be set to "requested". Additionally, if the value is set to "required" but the worker that performs preflight validation does not have exactly-once support enabled for source connectors, requests to create or validate the connector will fail.

	 - Type: STRING
	 - Default: requested
	 - Importance: MEDIUM
	 - Required: false

🔘 transaction.boundary

Permitted values are: poll, interval, connector. If set to 'poll', a new producer transaction will be started and committed for every batch of records that each task from this connector provides to Connect. If set to 'connector', relies on connector-defined transaction boundaries; note that not all connectors are capable of defining their own transaction boundaries, and in that case, attempts to instantiate a connector with this value will fail. Finally, if set to 'interval', commits transactions only after a user-defined time interval has passed.

	 - Type: STRING
	 - Default: poll
	 - Importance: MEDIUM
	 - Required: false

🔘 transaction.boundary.interval.ms

If 'transaction.boundary' is set to 'interval', determines the interval for producer transaction commits by connector tasks. If unset, defaults to the value of the worker-level 'offset.flush.interval.ms' property. It has no effect if a different transaction.boundary is specified.

	 - Type: LONG
	 - Default: null
	 - Importance: LOW
	 - Required: false

 Azure Service Bus
==========================
🔘 azure.servicebus.sas.keyname

Azure Service Bus Shared Access Signature Key Name

	 - Type: STRING
	 - Default: null
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.sas.key

Azure Service Bus Shared Access Signature Access Key

	 - Type: PASSWORD
	 - Default: null
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.namespace

Azure Service Bus Namespace that message entity belongs to.

	 - Type: STRING
	 - Default: null
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.entity.name

Azure Service Bus messaging entity name. Should be either the name of the queue or topic from which messages have to be polled.

	 - Type: STRING
	 - Default: null
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.subscription



	 - Type: false
	 - Default: STRING
	 - Importance: The Azure Service Bus Subscription Name for the configured topic from which messages have to be polled. This should only be configured when message entity is of type topic.
	 - Required: HIGH

🔘 azure.servicebus.connection.string

Fully qualified Azure Service Bus Connection string for ``Message Entity`` from which this source connector will poll messages. Only in the case where ``Message Entity`` is ``topic``, set the ``azure.servicebus.subscription`` config to the topic's subscription name. To establish connection with Azure Service Bus, either set all of these properties, azure.servicebus.sas.keyname, azure.servicebus.sas.key, azure.servicebus.namespace, azure.servicebus.entity.name or simply set azure.servicebus.connection.string.

	 - Type: PASSWORD
	 - Default: null
	 - Importance: HIGH
	 - Required: false

🔘 azure.servicebus.max.message.count

Maximum number of messages to be pulled in a single poll from the configured Service Bus message entity.

	 - Type: INT
	 - Default: 50
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.max.waiting.time.seconds

Maximum server wait time until which connector will try to pull records from message entity.

	 - Type: INT
	 - Default: 30
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.max.retry.time.seconds

The maximum time until which connector will retry pulling records with exponential backoff, in the case of exception being raised.

	 - Type: LONG
	 - Default: 5
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.max.auto.lock.renew.duration.minutes

The maximum duration in minutes for which the lock on the message will be automatically renewed. This ensures that the message lock does not expire while the message is being processed.

	 - Type: INT
	 - Default: 10
	 - Importance: LOW
	 - Required: false

🔘 azure.servicebus.prefetch.count

The prefetch count for the Service Bus receiver. Prefetching messages increases throughput by retrieving messages from the service before they are requested by the application. Setting this value to 0 disables prefetching.

	 - Type: INT
	 - Default: 0
	 - Importance: LOW
	 - Required: false
